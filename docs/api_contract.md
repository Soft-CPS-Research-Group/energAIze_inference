# API Contract Reference

This document is the source of truth for runtime HTTP contracts and payload routing behavior.

## API Surface

- `POST /admin/load`
- `POST /admin/unload`
- `POST /inference`
- `GET /info`
- `GET /health`

Notes:

- A single deployment can load one or many agents from one manifest.
- Each inference request resolves exactly one agent decision.
- Request body supports nested objects; the service flattens keys internally.

## POST /admin/load

Load a manifest and all agent artifacts in that manifest. Optionally select the default agent.

Request body fields:

- `manifest_path` (`string`, required): path to `artifact_manifest.json`.
- `agent_index` (`integer`, optional): default agent index for requests without `agent_index`.
- `artifacts_dir` (`string`, optional): root directory for artifact paths; defaults to manifest directory.
- `alias_mapping_path` (`string`, optional): path to alias JSON map.

Single-agent example:

```json
{
  "manifest_path": "/data/icharging_boavista_with_flex/artifact_manifest.json",
  "agent_index": 0,
  "artifacts_dir": "/data/icharging_boavista_with_flex",
  "alias_mapping_path": "/data/icharging_boavista_with_flex/aliases.json"
}
```

Multi-agent example with explicit default:

```json
{
  "manifest_path": "/data/icharging_community_boavista_sao_mamede/artifact_manifest.json",
  "agent_index": 1,
  "artifacts_dir": "/data/icharging_community_boavista_sao_mamede",
  "alias_mapping_path": null
}
```

Success response (`200`):

```json
{
  "status": "loaded",
  "manifest_path": "/data/icharging_community_boavista_sao_mamede/artifact_manifest.json",
  "agent_index": 1,
  "default_agent_index": 1,
  "loaded_agent_indices": [0, 1],
  "loaded_at": "2026-02-22T12:00:00.000000Z"
}
```

Other responses:

- `404`: manifest or alias path not found.
- `500`: generic load failure.

## POST /admin/unload

Unload currently configured pipelines.

Success response (`200`):

```json
{
  "status": "unloaded"
}
```

Other responses:

- `409`: no model is currently configured.

## POST /inference

Run inference for one agent.

Request fields:

- `agent_index` (`integer`, optional): target agent index. If omitted, default agent from `/admin/load` is used.
- `features` (`object`, required): feature payload.

Response shape:

- `actions`: object keyed by agent index string, value is action map.
- `request_id`: optional request identifier propagated from `x-request-id` or generated by middleware.

Example response:

```json
{
  "actions": {
    "1": {
      "BB000SMI": 22.4,
      "virtual_battery_kw": 10.0
    }
  },
  "request_id": "8bca4cc67c5c49f4b7451777fa0bd7bb"
}
```

Use direct payload under `features`.

```json
{

    "non_shiftable_load": 4.0,
    "solar_generation": 1.5,
    "energy_price": 0.1,
    "charging_sessions": {
      "AC000004_1": {
        "power": 1.8,
        "electric_vehicle": "11824"
      }
    },
    "electric_vehicles": {
      "11824": {
        "SoC": 0.3,
        "flexibility": {
          "estimated_soc_at_departure": 0.8,
          "estimated_time_at_departure": "2026-02-22T14:00:00Z"
        }
      }
    }
  }
}
```

### Inference Mode B: community features by site

Use `features.sites.<input_site_key>`. Routing key is read from artifact config `input_site_key`.

With explicit agent:

```json
{
  "agent_index": 1,
  "features": {
    "timestamp": "2026-02-22T12:00:00Z",
    "sites": {
      "boavista": {
        "non_shiftable_load": 7.0,
        "solar_generation": 2.0
      },
      "sao_mamede": {
        "site": {
          "pt_available_kw": 80.0
        },
        "solar_generation": 12.0,
        "charging_sessions": {
          "BB000SMI": {
            "power": 0.0,
            "electric_vehicle": "SM_EV_01"
          }
        },
        "electric_vehicles": {
          "SM_EV_01": {
            "SoC": 0.35,
            "flexibility": {
              "estimated_soc_at_departure": 0.8,
              "estimated_time_at_departure": "2026-02-22T16:00:00Z"
            }
          }
        },
        "electrical_storage": {
          "soc": 0.5
        },
        "virtual_battery": {
          "setpoint_kw": 18.0
        }
      }
    },
    "community": {
      "price_signal": 0.2
    }
  }
}
```

Without `agent_index` (uses loaded default):

```json
{
  "features": {
    "timestamp": "2026-02-22T12:00:00Z",
    "sites": {
      "boavista": {
        "non_shiftable_load": 7.0,
        "solar_generation": 2.0
      },
      "sao_mamede": {
        "site": {
          "pt_available_kw": 80.0
        },
        "solar_generation": 12.0
      }
    }
  }
}
```

Timestamp fallback behavior:

- If selected site payload does not include `timestamp`, top-level `features.timestamp` is copied in.
- If `timestamp` and `timestamp.$date` are both absent at site level, top-level `timestamp.$date` is also considered.

## GET /info

Returns metadata for the currently loaded default pipeline context.

Success response (`200`) example:

```json
{
  "algorithm": "RuleBasedCommunity",
  "manifest_version": 1,
  "metadata": {
    "experiment_name": "icharging_community_boavista_sao_mamede"
  },
  "topology": {
    "num_agents": 2
  },
  "service_version": "0.1.0",
  "agent_index": 1,
  "default_agent_index": 1,
  "loaded_agent_indices": [0, 1],
  "action_names": ["BB000SMI", "virtual_battery_kw"],
  "uptime_seconds": 56.91,
  "loaded_at": "2026-02-22T12:00:00.000000Z",
  "alias_mapping_path": null
}
```

Notes:

- `action_names` corresponds to the default agent (`default_agent_index`) action contract.
- If no model is configured, response is `503`.

## GET /health

Always returns `200` with readiness and deployment metadata.

Example:

```json
{
  "status": "ok",
  "configured": true,
  "agent_index": 1,
  "default_agent_index": 1,
  "loaded_agent_indices": [0, 1],
  "providers": ["rule_based"],
  "manifest_path": "/data/icharging_community_boavista_sao_mamede/artifact_manifest.json",
  "alias_mapping_path": null,
  "gpu_available": false
}
```

## Error Semantics

`POST /inference`:

- `400`:
  - invalid/missing community site object for selected `input_site_key`;
  - unknown `agent_index` requested;
  - other runtime `RuntimeError` cases not related to configuration state.
- `503`: pipeline not configured.
- `500`: unexpected execution failure.

`POST /inference` missing-site example (`400`):

```json
{
  "detail": "\"Missing required object 'features.sites.sao_mamede' for agent_index 1\""
}
```

`GET /info`:

- `503`: pipeline not configured.

`POST /admin/load`:

- `404`: path not found.
- `500`: any other load exception.

`POST /admin/unload`:

- `409`: no model configured.

## Multi-Agent Routing

Routing algorithm per request:

1. Resolve target agent:
2. If request has `agent_index`, use it.
3. Otherwise use default agent loaded by `/admin/load`.
4. If target artifact config has `input_site_key`, payload must provide `features.sites.<input_site_key>`.
5. Selected site payload is flattened and sent to runtime.
6. Exactly one agent result is returned in `actions` map for that request.

## iCharging Operational Semantics

These points are intentional in current operation and reflect hardware requirements.

1. A charger may receive a technical minimum setpoint even if no EV is physically connected.
2. For standard AC connectors this technical minimum is commonly `1.6 kW`.
3. For tri-phase BB chargers, technical minimum may be `8.0 kW` total, distributed as one-third per phase.
4. In controller totals used for operational limits (`board_total`), disconnected chargers can be excluded by current logic.
5. When EV is connected, dispatch is constrained to charger min/max and line or board limits.

This behavior is specific to configured runtime strategy and manifest parameters. It is not a generic electrical model.

## Operational Note: Admin Endpoint Security

Current service implementation does not enforce built-in authentication or authorization on `/admin/*`.

For production deployments:

1. Put the service behind an authenticated gateway or reverse proxy.
2. Restrict network access to admin routes.
3. Use environment-level controls (private subnet, ACLs, service mesh policies).
